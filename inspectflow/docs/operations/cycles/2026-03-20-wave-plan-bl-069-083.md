# Wave Plan: BL-069 Through BL-083

## Scope
Execution sequencing for the newly accepted capability expansion + simplification backlog set.

## Wave 1 (Immediate Foundation)
1. `BL-069` Incremental near-real-time analytics mart updates.
2. `BL-070` Supervisor/admin performance analytics tab.
3. `BL-071` SPC capability layer (`Cp`, `Cpk`, `Pp`, `Ppk` + control rules).

Rationale:
- `BL-069` is the freshness foundation required before high-trust supervisor/SPC surfaces.
- `BL-070` and `BL-071` directly deliver the highest requested analytics value after freshness.

## Wave 2 (Operator Workflow Depth)
1. `BL-072` Native parser pack (metrology formats).
2. `BL-073` Mobile/tablet-first operator UX.
3. `BL-074` Submission document attachments.
4. `BL-075` Versioned job/measurement instructions + acknowledgements.
5. `BL-076` Structured AS9102 FAI workflow.
6. `BL-077` Legacy frontend monolith migration + removal.

Rationale:
- These items deepen operator and quality execution once analytics freshness/reporting is stable.

## Wave 3 (Admin, Governance, and Surface Simplification)
1. `BL-078` Global cross-entity search.
2. `BL-079` Audit/compliance viewer with evidence exports.
3. `BL-080` Seat/module entitlement admin console.
4. `BL-081` Integration-layer simplification.
5. `BL-082` OIDC SSO standardization and legacy path deprecation.
6. `BL-083` Edge-sync de-defaulting / legacy mode.

Rationale:
- These are governance and platform-shape improvements that can proceed after operator + analytics foundations are active.
